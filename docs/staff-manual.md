# スタッフ運用マニュアル

本マニュアルは、AI Pose Analyzer を受付〜施術〜説明の現場フローに組み込むための実務手順を示す。

---

## 1. 利用フローの全体像

```
[受付] → [初回案内 / 同意取得] → [LIFF起動 → 撮影] → [施術] → [施術後撮影] → [Before/After 説明]
```

---

## 2. 各場面でのスタッフ対応

### 2.1 初回来院時（受付）

1. 公式 LINE 友だち追加が済んでいるか確認。未追加なら QR コードを案内する。
2. リッチメニューの **「姿勢チェック」** をタップしてもらい、同意画面に進む。
3. プライバシーポリシー（`docs/terms.md`）の要点を口頭で説明：
   - 「お写真は患者さんのスマホの中だけで処理され、外には出ません」
   - 「保存されるのはスコアと骨格の点の座標だけです」
   - 「いつでも削除できます（スタッフへ申し出ください）」

### 2.2 撮影時

- **立ち位置**: 壁から 1.5m 前、両足を肩幅で正面向き、リラックスして立つ。
- **服装**: 全身が見える服。コートや厚手の上着は脱いでもらう。
- **画角**: 頭頂部からつま先まで画面内に収まること。
- **背景**: できるだけシンプルな背景（壁・カーテン等）。
- **照明**: 顔と体に均等に光が当たる位置に立ってもらう。

### 2.3 結果表示

- スコアは 0〜100 点で表示される。
- 4 つの指標（肩の水平度・骨盤の水平度・頭部の傾き・体の中心軸）の内訳を提示。
- **絶対に「診断」「治療」と言わないこと**。
  - ❌「左肩が下がっています、骨盤が歪んでいます」
  - ⭕「左右の肩のバランスに少し差がある傾向ですね」

### 2.4 施術後

- 同じ立ち位置・服装・画角で再撮影してもらう。
- 履歴一覧から **施術前の項目をタップ** すると、Before/After が並んで表示される。
- 差分（▲ ポイント改善 / ▼ ポイント低下）が表示される。

### 2.5 経過観察（来院ごと）

- 毎回撮影を習慣にしてもらう。
- スコアの **絶対値より変化量・推移** を見て会話する。
- 月単位の推移はスタッフ側で `姿勢ツール → 選択ユーザーの推移を表示` で確認できる。

---

## 3. スタッフ側ツール（スプレッドシート）

### 3.1 起動

院の代表アカウントで保存先スプレッドシートを開くと、画面上部に **「姿勢ツール」** メニューが表示される。

### 3.2 ユーザー一覧（summary シート）

- 「ユーザー一覧を更新」を実行 → `summary` シートが作成/更新される。
- 列: userId / 記録数 / 初回日 / 初回スコア / 最終日 / 最終スコア / 変化量 / 最大 / 最小
- 並べ替え・フィルタはシート機能でそのまま利用可能。

### 3.3 個別ユーザーの推移グラフ

- `summary` シートで対象 userId のセル（A 列）を選択した状態で、「選択ユーザーの推移を表示」を実行。
- `trend` シートに折れ線グラフが描画される。

### 3.4 userId と実患者の紐付け

セキュリティ設計上、システム上には実名は **保存されない**。患者と userId の対応は紙カルテ or 別管理表で運用すること。

> 推奨運用: 紙カルテに `LINE: U1234...abcd（先頭8桁）` のメモを書いておく。

---

## 4. トラブルシューティング

| 症状 | 対処 |
| --- | --- |
| カメラが起動しない | LINE のブラウザ権限を許可。iOS は設定 → LINE → カメラ ON |
| 「姿勢を検出できませんでした」 | 全身が画面に入っているか、明るさ十分か確認 |
| 「通信エラー (HTTP xxx)」 | GAS の Web App URL が `config.js` と一致しているか、デプロイ最新か |
| 「IDトークン検証失敗」 | LIFF のチャネル ID と GAS の `LINE_CHANNEL_ID` が一致しているか |
| 履歴が他人の分まで見える | あり得ない（GAS は IDトークンの sub でフィルタリング）。報告 |
| 保存ボタンを押しても増えない | スプレッドシート ID が `SHEET_ID` と一致しているか、`records` シートが書き込み可能か |

---

## 5. 患者データの削除依頼

患者から「データを消してほしい」と申し出があった場合：

1. `summary` シートで対象 userId を確認。
2. `records` シートを userId 列でフィルタし、対象行を選択して削除。
3. `imageFileId` 列の値（Drive ファイル ID）を控え、Drive 上の対応ファイルも削除。
4. 必要に応じて `summary` を再生成。

---

## 6. 守るべきコンプライアンス

- 撮影の同意なしに記録を残さない（同意チェック前は LIFF が次画面に進まない設計）。
- スコア・所見は **「姿勢傾向の参考情報」** に留め、診断・治療を示唆しない。
- 解析結果のスクリーンショットを SNS 等で第三者に共有しない。
