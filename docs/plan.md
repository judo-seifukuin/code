# 姿勢評価システム（AI Pose Analyzer）開発・運用計画書【LIFF×サーバーレスAI 決定版】

## 1. プロジェクトの目的と背景

本プロジェクトは、10周年の節目を迎える本院において、施術効果の「可視化」を通じて患者の納得感を引き出し、継続的な来院を促進する次世代の姿勢評価システムを構築するものである。患者自身が日常的に利用するLINEを通じ、各自のスマートフォンで手軽に姿勢チェックと過去の比較（Before/After）ができる環境を提供し、自宅でのセルフケアのモチベーション向上と院とのエンゲージメント強化を図る。

## 2. 新システムアーキテクチャ（ハイブリッド構成）

サーバー維持費の削減と極めて高いセキュリティを両立するため、AI処理をクライアント（患者のスマホ）側にオフロードする「サーバーレスAIアーキテクチャ」を採用する。

| レイヤー | 採用技術・ツール | 役割と設計上のメリット |
| :-: | :-: | :-: |
| フロントエンド (UI) | LINE LIFF アプリ | LINEのリッチメニューからシームレスに起動。LINEログインの自動化により、患者の手間を省きつつセキュアにユーザーIDを取得する。 |
| 画像解析エンジン | ブラウザ版 MediaPipe Pose (JavaScript) | 患者のスマートフォン端末（ブラウザ）のCPU/GPUを用いてAI解析を完結させる。外部サーバー（Cloud Run等）が不要となるため、維持費が完全無料となり、通信遅延もない。 |
| バックエンド (API・制御) | Google Apps Script (GAS) | LIFFアプリから送信された解析データを受け取り、保存や履歴の呼び出しを行うハブとして機能する。 |
| データベース (DB) | Google スプレッドシート / Drive | 解析スコアと暗号化された画像データを厳格な権限下で管理する。 |

## 3. 徹底したデータ漏洩対策（最重要要件）

患者自身のスマートフォンを利用するシステムであるため、プライバシー保護と情報漏洩対策をシステムの根幹に据える。

- **生画像の外部送信禁止（エッジAI処理）:** 姿勢解析はすべて患者の端末内で完結する。解析後の画像はBase64形式の暗号化文字列に変換された上でバックエンドへ送信され、元画像がそのままインターネット上を流れるリスクを物理的に遮断する。
- **LINE IDトークンによる厳格な本人確認:** 過去の履歴データを要求する際、単純なLINE IDではなく、暗号化された「IDトークン」でのみ照合を行い、悪意のある第三者によるなりすましアクセスを完全に防ぐ。
- **個人情報（PII）の分離:** データベースには「LINEユーザーID（無機質な英数字）」と「姿勢データ」のみを記録する。システム上で実名などのカルテ情報は持たせず、万一の流出時にも個人を特定できない設計とする。
- **公開URLの発行禁止:** 保存先となるGoogleドライブは「非公開（院の管理者のみ）」とし、URLによる直接アクセスを禁止する。

## 4. 機能要件定義 (MVP)

### 4.1 患者向け機能

- **ワンタップ起動・撮影:** リッチメニューから起動し、アウトカメラで即座に撮影（自動で解析に最適なサイズへ軽量化）。
- **リアルタイムAI解析:** 撮影直後に画面上で骨格ランドマーク（33点）を検出し、姿勢スコアを算出。
- **Before/After比較:** 過去の履歴を呼び出し、初診時や前回時の姿勢と現在の姿勢を画面上に並べて比較・確認できる機能。

### 4.2 院内運用・スタッフ向け機能

- **カルテ連携補助:** スプレッドシート上に蓄積されたデータ群から、必要な患者の推移を一覧で確認できる管理ビューの提供。

## 5. 運用プロトコルと現場定着化

- **院内テストの徹底:** 鈴木、小瀬野、豊田ら現場の施術スタッフ主導で、実際の業務フロー（受付〜施術〜説明）の中で無理なくアプリが案内・活用できるかを検証し、患者への案内マニュアルを構築する。
- **法的コンプライアンスの遵守:** 患者へ表示するスコアやアドバイス文言は「診断・治療」を暗示せず、「姿勢傾向の参考情報」に留める。利用開始時にはLINE上で必ずプライバシーポリシーへの同意を取得する。

## 6. 開発スケジュール・マイルストーン

| フェーズ | 主な実施内容 |
| :-: | :-: |
| **Phase 0: 計画承認・環境準備** | LINE DevelopersでのLIFFチャネル作成、GASプロジェクトの立ち上げ、データ保存用フォルダの権限設定。 |
| **Phase 1: フロントエンド開発 (LIFF)** | HTML/JSによるカメラ起動、MediaPipeによるブラウザ内解析、画像のBase64暗号化と軽量化処理の実装。 |
| **Phase 2: バックエンド開発 (GAS連携)** | LIFFからの暗号化データ受信、スプレッドシートへの安全な書き込み、IDトークンを用いた履歴データの呼び出し機能の実装。 |
| **Phase 3: 現場テスト・UX調整** | スタッフ間での模擬テスト（UIの使い勝手、エラーハンドリングの確認）、患者向け利用規約の作成。 |
| **Phase 4: 本番稼働** | リッチメニューへの正式実装と、患者への提供開始。 |
